Spotlight Rules

An openly-governed, progressive build of the Spectral API linter — a maintained CLI, a standalone ruleset specification, a VS Code extension, and a browser governance playground, under API Commons.

spotlight-spec

Specification

A standalone specification of the ruleset format — rules and rulesets — with a JSON Schema for validation, and an explorable catalog of every rule by artifact.

Details → GitHub
spotlight-cli

CLI & engine

The linter for JSON/YAML API descriptions — OpenAPI, AsyncAPI, and Arazzo — with custom rulesets. A maintained fork of Spectral, published to npm.

Details → GitHub
spotlight-vscode

Editor

The VS Code extension — lint-on-save and lint-on-type for your API descriptions, with custom ruleset support, right in your editor.

Details → GitHub
spotlight-validator

Playground

A browser governance playground — lint 11 artifact types against best-of-breed rules, search APIs.io, and edit rules live. No backend.

Details → Open
spotlight-api

HTTP API

Work with Spotlight over HTTP — lint artifacts, list rulesets, validate a ruleset. OpenAPI-first, runs the cli engine.

spotlight-mcp

AI · MCP

Govern your APIs conversationally — a Model Context Protocol server that lets any AI client lint artifacts and work with rulesets.

Details → GitHub
spotlight-pipeline

CI · Governance

Governance as code — run Spotlight as a gate in GitHub, GitLab, Bitbucket, Azure, or AWS pipelines. Drop in your ruleset.
