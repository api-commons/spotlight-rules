Spotlight Rules

An openly-governed, well-maintained build of the Spectral API linter — a maintained CLI, a standalone ruleset specification, a VS Code extension, and a browser governance playground, under API Commons.

spotlight-cli

CLI & engine

The linter for JSON/YAML API descriptions — OpenAPI, AsyncAPI, and Arazzo — with custom rulesets. A maintained fork of Spectral, published to npm. 

npm i -g @spotlight-rules/spotlight-cli
GitHub npm
spotlight-spec

Specification

A standalone specification of the ruleset format — rules and rulesets — with a JSON Schema for validation, independent of any single linter implementation. 

JSON Schema · draft-07 · versioned
GitHub Schema
spotlight-vscode

Editor

The VS Code extension — lint-on-save and lint-on-type for your API descriptions, with custom ruleset support, right in your editor. 

Extension id: api-commons.spotlight
GitHub
spotlight-validator

Playground

A browser governance playground — lint OpenAPI, AsyncAPI, and JSON Schema with best-of-breed rules compiled from public rulesets. Select rules by tag, or edit your own. No backend.
